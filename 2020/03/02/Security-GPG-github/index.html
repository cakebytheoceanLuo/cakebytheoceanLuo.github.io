引入

在本地的 git 仓库中，我们需要设置用户名和邮箱地址才能提交新的更改。Pro Git这本书中提到：设置用户名和邮件地址是我们下载完git这个软件后必须做的第一件事情。

值得注意的是，这并不需要跟你的 GitHub 账号相同。这带来了一个很严重的问题：任何人都可以设置用户名和邮箱地址来伪装成你，由于 GitHub 会自动根据用户名和邮箱地址关联单次的提交为注册的 GitHub 的账户，所以别人可以仿冒你的身份进行提交更改。这带来了一个很严重的问题：任何人都可以设置用户名和邮箱地址来伪装成你，由于 GitHub 会自动根据用户名和邮箱地址关联单次的提交为注册的 GitHub 的账户，所以别人可以仿冒你的身份进行提交更改。1

GitHub使用GPG这个加密方法来对commit进行验证，来确定本次commit的标签(tag)是由gpg密钥所有者所提交的。这种被验证过的commit会带有一个很酷同时很安全的verfified标签：

而没有该verified标签未必是本人提交，因为github的提交log中的提交人可以随意修改。

添加 public key 到 GitHub

公钥(public key)文件().gnupg/pubring.gpg)以二进制形式储存，--armor参数可以将其转换为ASCII码显示。

  • 我们需要以standard out的方式输出我们的公钥: gpg --armor --export [Key ID]
  • 或者我们加上--output <filename>参数，可以将这个公钥写入一个输出文件: gpg --armor --output public-key.txt --export [Key ID]

补充注释

2:

这里我们使用的前提是git config --global，即对当前用户 进行设置。如果你的想法只是在一个当前项目中使用gpg key来验证，那就在当前项目的文件夹中，输入以上命令不带--global。比如git config commit.gpgsign true。

这里我主观地多一句嘴，gpg是一个很安全可靠的工具，它值得我们使用--global这个flag。

然后我们复习一下Pro Git中的一点有关内容：

git配置保存在3处，对应着三个级别。配置优先级为: 本项目 > 当前用户 > 系统全局:

作用范围 配置文件 git config选项
系统全局 /etc/gitconfig --system
当前用户 ~/.gitconfig或~/.config/git/config --global
本项目 项目目录/.git/config 无flag
