[Security]GPG介绍与密钥生成

发表于 2020-03-02 更新于 2020-03-03 分类于 Operating System 阅读次数：
本文字数： 4.8k 阅读时长 ≈ 4 分钟

公开密钥密码学(Public-key cryptography)或非对称式密码学(Asymmetric cryptography)是一种非常流行的加密算法，它应用于去中心化的信任网络以及操作系统中。这个算法在现实中对信息加密和解密，可以用到GnuPG软件(简称GPG)，它是目前最流行的加密工具之一。本文主要介绍gpg软件的使用和生产密钥对。本文的使用环境为Linux Ubuntu命令行。另外在这篇文章中我使用的是gpg (GnuPG) 2.2.4. libgcrypt 1.8.1.　Copyright (C) 2017 Free Software Foundation, Inc.的英文版本。


GPG介绍

要了解什么是GPG，就要先了解PGP(Pretty Good Privacy)。

1991年，程序员Phil Zimmermann为了避开政府监视，开发了加密软件PGP。这个软件非常好用，迅速流传开来，成了许多程序员的必备工具。但是，它是商业软件，不能自由使用。所以，自由软件基金会决定，开发一个PGP的替代品，取名为GnuPG。这就是GPG的由来。1

这个问题是让用户自己选择加密算法。可以直接按Enter回车键选择默认第一个选项，表示加密和签名都使用RSA算法。


然后出现一个问题，问用户密钥的长度。最好我们输入最长的4096，然后按Enter回车键。


接着我出现一个问题，要求设置密钥的有效期(valid time length)。如果密钥只是个人使用，并且确认可以有效保管私钥，建议选择第一个选项，即永不过期(expire)。输入0，然后按Enter回车键。


回答完上面三个问题以后，需要确认。在确认以后，输入y，然后按Enter回车键。
