Advanced SQL - 高级SQL

Slide: https://15445.courses.cs.cmu.edu/fall2019/slides/02-advancedsql.pdf
Note: https://15445.courses.cs.cmu.edu/fall2019/notes/02-advancedsql.pdf
Reading: Chapters 3-5

这节课，我们会了解高级的SQL句法。(但是实际上这门课对SQL的要求很低，大量其他的SQL练习依旧可以在我的博客中找到)

如果大家的目的是为了学习SQL，我建议使用PostgreSQL，它出自于学术界，整体行为相对MySQL和SQLite更合理。另外TUM和CMU在SQL教学上都采用PostgreSQL。

我将这次笔记分成前后两个部分，这是第一部分。

SQL(a.k.a. Strutured Query Language)并不是这门语言一开始名字，IBM给这个语言的最初的名字是　SEQUEL。所有现在大部分人依旧将　SQL 读作　SEQUEL。

最好能讲上面几个概念记住，我们学校在本科生的数据库课中考SQL的全称，百分之九十的同学回答错误。

Comparison of different SQL implementations：　http://troels.arvin.dk/db/rdbms/

最好大致对SQL的标准有一定的印象，因为各个RDBMS对SQL标准的支持都不一样，大有百花齐放之势，我们会在这节课后面的部分见到。

这里要注意: SQL使用的是bag semantic，即允许重复值出现。

check是确认insert指是否满足我们规定的不变量(invariant):

on delete cascade

cascade: 很象征地描述了一种像瀑布一样的连锁效应。这个词在西方语境下比较常用，对应我们可以理解成连锁效应即可。(这个词这个门课最后的事务transaction还会出现。)

enrolled这个表格的字段都是来自student和course的外键，因此这个表格依赖于其他两个表格。让我们想象一下，如果我们删去一个student或course，如果删去的tuple在enrolled拥有外键，那么对应在enrolled的tuple也需要被删除。不然enrolled中就出现了一个僵尸：对应不到student或course。

HAVING

HAVING从句可以说是WHERE从句带聚合函数的版本。它们俩都在对tuple做一些筛选的工作。后面我们会在query optimization中提到logical plan中的selection pushdown / early filtering。这个优化就是尽可能早得对提前筛选出符合WHERE和HAVING从句条件的tuple，尽量让查询中间结果的集合尽可能小。
